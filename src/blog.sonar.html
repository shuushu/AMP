SonarQube 정적분석 툴 설치 및 사용하기

2017-08-22 / by sk컴즈 UIDEV

CSS, JS, HTML 문법 검사 SonarQube! 적은 시간과 비용으로 노력을 줄이고 품질을 향상 시킨다.

1-1. Java 설치하기

아래의 링크를 통해서 최신 버전의 Java를 설치하세요. http://java.com/ko/download/

2-1. SonarQube 설치하기.

http://www.sonarqube.org/ ◀링크에서 SonarQube를 다운로드 받습니다.
sonarqube/conf/sonar.properties를 수정합니다
mysql을 DB로 사용하기때문에 mysql쪽 주석만 풀어서 사용하면 됩니다.
url주석을 푼후, 알맞게 username과 password를 수정해줍니다.
이때 username과 password는 mysql 접속 정보를 입력합니다.
그리고 mysql 에 "sonar" schema가 생성되어 있어야 정상적으로 연결됩니다.

2-2. Sonar Runner 다운로드

https://docs.sonarqube.org/display/SONARQUBE52/Installing+and+Configuring+SonarQube+Scanner
다운로드 받은 zip 파일을 원하는 경로를 지정하여 압축을 해제합니다.
<설치 디렉토리>/conf/sonar-runner.properties 파일을 열어서 자신의 환경에 맞게 수정합니다.
동일한 PC에 SonarQube를 설치했고 기본 설정을 변경하지 않았으면, sonar.jdbc.* 속성의 주석을 해제하는 정도가 할일의 전부입니다.
저는 SonarQube 서버를 같은 PC에 설치했고 MySQL을 사용하고 있기 때문에 아래과 같이 설정을 완료했습니다.
SONAR_RUNNER_HOME 시스템 환경변수를 만들고 그 값으로 [설치 디렉토리]를 설정합니다.
[설치 디랙토리]\bin 디렉토리(%SONAR_RUNNER_HOME%\bin)를 path에 추가합니다. 이제 SonarQube Runner의 설치가 완료되었습니다.

2-3 소나큐브 실행후 열어보기

소나큐브 디렉토리 / bin/폴더내에 자신에 맞는 운영체제 폴더로 이동합니다. 후 bat파일을 돌리시면 프롬트프가 뜨고, 실행됩니다.

그후 설정한 호스트와 포트로 접속하여 데시보드가 뜨는지 확인합니다.

프로젝트 분석

1-1 필요한 플러그인 추가

sonarqube/extensions/plugins 폴더아래 플러그인을 넣습니다.
제가 추가한 플러그인을 첨부합니다. plugins.zip (추가후 소나큐브 재실행)

1-2 rule import

우리팀에서 정해놓은 룰을 import 합니다
Configuration --> Quality Profiles
Restore Profile (우측 상단에 create옆 하단화살표표시클릭, 로그인 해야 보입니다. 계정 : admin/admin)
하단 export 파일들을 불러옵니다 (총6개)

  • web-sonar-way-09975.xml
  • less-sonarqube-way-37269.xml
  • json-sonarqube-way-25598.xml
  • js-sonar-way-09262.xml
  • java-sonar-way-75002.xml
  • css-sonarqube-way-28648.xml

1-3 sonar-runner 실행

A. 소나러너를 돌릴 산출물 디렉토리에 sonnar-project.properies 파일(소나러너디렉토리에 sample-sonar-project.properies 복사후 이름바꿔서 사용)을 넣고, Key값 projectName, Version등을 입력합니다.

그외의 추가 설정 값 :
sources : 소스를 돌릴 폴더, 파일을 지정합니다. 모두 돌리고 싶을땐 . 으로 설정
sources : 돌리지 않을 소스, 제외소스를 지정합니다.(라이브러리나 테스트 코드 등..)

B. 프롬트프(cmd)로 해당 디렉토리에 이동(cd /디렉토리위치) 하여 sonar-runner를 실행합니다(SonarQube 서버는 이미 동작 중이어야 합니다).
